Инструкция SOC-аналитика
  1. Очередь NEW — приоритизируется по severity. Берите сначала Critical (sev 5), затем High (sev 4).
  2. «Взять» — назначает инцидент на вас и переводит в IN_PROGRESS.
  3. Решить (зелёный ) — закрытие инцидента после обработки. Можно добавить комментарий через карточку.
  4. Ложное срабатывание (серый ) — корреляция ошиблась, действия не требуются.
  5. Открыть карточку (оранжевый ) — детальный просмотр, передача другому аналитику, связанные события аудита.
