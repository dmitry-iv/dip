Генератор демо-данных

Создаёт реалистичную картину активности SOC за последние 24 часа. Используется для презентаций, тестирования UI и проверки работы корреляционных правил.

Сгенерировать демо-данные

Будет создано:

  • ~25 успешных логинов от разных пользователей
  • ~18 операций доступа к документам
  • ~8 неудачных попыток входа
  • ~5 административных действий (изменение ролей)
  • ~4 операции экспорта данных
  • 6 инцидентов разной серьёзности:
    • BRUTE_FORCE (Critical, sev 5)
    • CREDENTIAL_STUFFING (High, sev 4)
    • MFA_BYPASS_ATTEMPT (High, sev 4)
    • PRIVILEGE_ESCALATION (High, sev 4)
    • MASS_EXPORT (Medium, sev 3)
    • OFF_HOURS_LOGIN (Medium, sev 3)
Если включены email-алерты, на инциденты severity ≥ 4 будут отправлены письма. При необходимости отключите алерты в настройках перед генерацией.
Очистить демо-данные

Удаляет только записи с маркером [DEMO] в полях details/description. Реальные пользовательские записи остаются нетронутыми.

Внимание. Удаление записей из audit_log нарушает hash-цепочку — это штатное поведение tamper-evident журнала: любая модификация исторических записей детектируется. Если после очистки требуется полностью сбросить цепочку, выполните в pgAdmin: 
TRUNCATE TABLE audit_log;
UPDATE audit_chain_state SET last_hash='GENESIS', last_seq=0 WHERE id=1;
Назначение

Генератор позволяет наполнить систему реалистичными данными для:

  • Демонстрации работы дашборда, журнала аудита и списка инцидентов на наполненной системе
  • Тестирования корреляционных правил и карточек инцидентов разной серьёзности
  • Проверки tamper-detection: после генерации можно намеренно модифицировать запись в БД (например, UPDATE audit_log SET details='...' WHERE ...) и убедиться что система детектирует разрыв цепочки
  • Отладки UI — графиков, фильтров, цветовой индикации severity

Все демо-записи помечаются маркером [DEMO] и могут быть удалены одной кнопкой.
